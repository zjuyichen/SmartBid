When Forgetting Matters
More Than Remembering

The First Global Agentic Tender Generation Challenge. Can your AI Agent actively purge Toxic Memories to survive regulatory shifts and cost spikes?

Application Background: Automated Tender Generation

SmartBid challenges AI agents to automatically generate professional tenders/bids for government and enterprise procurement projects. In real-world scenarios, tender documents often change due to:

  • New regulatory requirements (e.g., GDPR, local compliance laws)
  • Raw material price fluctuations that invalidate previous pricing strategies
  • Technical standard updates making old specifications obsolete

The core challenge: Can your Agent forget outdated winning strategies and adapt to new requirements without retraining?

The ASE Evaluation Protocol

Simulating the lifecycle of a B2B Agent: From Accumulation to Shift to Recovery.

PHASE A

Memory Accumulation

Agent processes 30 historical tender projects to build initial memory $\mathcal{M}_{A}$.

  • 60% High-utility Cases
  • 20% Suboptimal Strategies
PHASE B

Context Shift

Environmental rules change. Previously successful strategies become Toxic Memories.

Example: GDPR Update Invalidates Old Data Clauses.
PHASE C

Long-Horizon Eval

Execute 50 tasks under new rules. No Fine-tuning Allowed. Survival depends on memory management.

Recovery Speed Target: High

Core Evaluation Metrics

NTR

Lower is Better

Negative Transfer Rate

The proportion of task failures directly attributable to the retrieval of stale or "toxic" memories.

RS

Higher is Better

Recovery Speed

Number of steps required after Context Shift to restore performance to 90% of peak.

FIS

Higher is Better

Forgetting Stability

Measures the variance of performance after forgetting actions. Higher values indicate controlled forgetting.

MSG

Lower is Better

Memory Size Growth

Rate of memory store growth. We penalize "memory bloat" to encourage efficient abstraction.
