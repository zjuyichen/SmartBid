当 遗忘 比记忆更重要

首届智能体标书生成大赛。您的 AI Agent 能否在“法律法规变更”或“原材料价格暴涨”后， 主动遗忘 (Active Forgetting) 过时的成功经验，生成符合新环境的完美标书？

为什么要挑战“遗忘”？

在传统的 RAG (检索增强生成) 架构中，我们假设“记忆越多越好”。但在标书生成这种高风险领域，这会导致严重问题：

  • [cite_start]记忆毒性 (Memory Toxicity) [cite: 60] 去年的“低价策略”在今年原材料涨价后，如果被 AI 检索并引用，将直接导致亏损废标。
  • [cite_start]错误放大 (Error Amplification) [cite: 72] 一次侥幸中标的违规条款，可能被 Agent 当作“黄金样本”反复学习。
  • [cite_start]无限增长成本 [cite: 74] 无差别的记忆存储会导致推理延迟激增，拖慢投标响应速度。

ASE 评估协议

[cite_start]

本次比赛严格遵循论文提出的三阶段评估法 [cite: 62]

PHASE A

Memory Accumulation
(记忆积累期)

Agent 需处理 30 个历史招标项目。数据包含混合质量样本：

  • 60% 优质中标案例
  • 20% 次优案例 (埋藏隐患)
  • 20% 包含过时条款

Goal: Build $\mathcal{M}_{A}$

PHASE B

Context Shift
(规则突变期)

模拟真实环境突变。触发 Context Shift 事件，使得 Phase A 中积累的部分记忆瞬间变为 Toxic Memory。

事件示例：新隐私法颁布，所有旧数据处理条款失效。
PHASE C

Long-Horizon Evaluation
(长程评估期)

在新规则下连续完成 50 个标书生成任务。 禁止微调模型，Agent 必须依靠 `forget()` 和 `abstract()` 机制来适应新环境。

核心评分指标 (Evaluation Metrics)

NTR

[cite_start]

Negative Transfer Rate [cite: 446]

负迁移率。在新环境下，Agent 错误引用“旧规则”（有毒记忆）导致废标的比例。

Lower is Better

RS

[cite_start]

Recovery Speed [cite: 446]

恢复速度。发生规则突变后，Agent 需要多少个步骤才能将胜率恢复到峰值的 90%？

Faster is Better

FIS

[cite_start]

Forgetting-Induced Stability [cite: 518]

遗忘诱导稳定性。衡量 Agent 在执行遗忘操作后，性能表现的方差。是否更加稳定？

Higher is Better

MSG

[cite_start]

Memory Size Growth [cite: 464]

记忆增长率。我们惩罚无脑堆砌数据的行为，奖励“小而美”的高效记忆库。

Lower is Better
[cite_start]

FINAL SCORE FORMULA [cite: 564]

Score = 0.3·AUC + 0.25·(1-NTR) + 0.2·UMR + 0.15·FIS + 0.1·(1-MSG)
